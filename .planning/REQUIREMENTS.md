# Requirements: Express Dynamic Surcharge Orchestrator — Milestone v1.1

**Defined:** 2026-05-10
**Milestone:** v1.1 Real-World Routing & Demo Hardening
**Core Value:** The agent must transparently reason through fuel price, route, and shipping data to produce an accurate, explainable surcharge recommendation — visible reasoning is what makes this agentic, not just automated.

> v1.0 requirements (43/43 validated) are archived at `.planning/milestones/v1.0-REQUIREMENTS.md`.
> This file tracks v1.1-only scope. New REQ-IDs continue v1.0 numbering where the category existed (DATA-06+); new categories (HUB, THEME, GUARD, PRICE, FIX) start at -01.

## v1.1 Requirements

Requirements for milestone v1.1. Each maps to exactly one roadmap phase. Items already shipped via quick tasks or debug fixes are marked `[x]` and reference the shipped artifact.

### Hub Network (HUB) — Phase 9 / 999.9

- [x] **HUB-01**: 10 hubs (1 HQ + 9 branches) seeded from `data/raw/hubs.json`, distributed by zone density (central-1: 5, central-2: 3, central-3: 2)
- [x] **HUB-02**: User can pick origin hub via HubPicker dropdown in chat UI; default selection on cold start = HQ Lat Krabang
- [x] **HUB-03**: User can override origin via prose for a single turn ("ship from Bang Na to Nonthaburi")
- [x] **HUB-04**: When origin is unspecified in both dropdown and prose, agent silently defaults to HQ and Pricing Agent narrates the assumption in a reasoning bullet
- [x] **HUB-05**: `lookup_rate` extended to `(shipping_type, origin_zone, dest_zone, weight_kg)` signature; resolves `origin_hub_id → origin_zone` via `hubs.json` map
- [x] **HUB-06**: Rate table extended to 135-row origin×destination matrix (3 origin-zones × 3 dest-zones × 3 ship types × 5 weight tiers) generated by extended `data/scripts/generate_rate_table.py`
- [x] **HUB-07**: Planner extracts `origin_hub_id` from user message with allowlist validation; falls back to dropdown's current value on invalid/missing
- [x] **HUB-08**: Route TTL cache key extended to `(origin_hub_id, destination)`; follow-up turn inheritance (Phase 5 D-08) extended to inherit `origin_hub_id` across turns

## v2 Requirements

Captured from 999.9 deferred ideas and 999.10/999.11 deferral notes; tracked but not in current roadmap.

- **NETW-01**: Regional expansion — Northeast / North / South EPPO baselines, expanded zones (deferred from v1.1; tracked as 999.8 backlog)
