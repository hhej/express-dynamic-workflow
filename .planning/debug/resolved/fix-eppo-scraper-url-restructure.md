## Resolution

root_cause: EPPO restructured the petroleum-statistics page sometime between 2026-04-03 (seed CSV cutoff) and 2026-05-09. The Joomla menu item slug was renamed from `en-petroleum-statistic` to `petroleumprice-statistic`. The published Excel files are now organized as Tables P03..P09, hosted under `/epposite/images/Energy-Statistics/energyinformation/Energy_Statistics/Petroleum_Prices/`. Table 9 (`P09.xls`) is "Retail Price of Petroleum Products in Bangkok" — the appropriate source for diesel B7 retail. Additionally the table is monthly (MIN / WT.AVG / MAX rows per year per month), pivoted across 21 columns; the original `_scrape_eppo()` `pd.read_excel` + simple column rename strategy cannot parse it. xlrd was also missing from the venv, blocking legacy .xls reads. P09 alone is insufficient for "today's freshness" (it updates monthly with a few-day lag), so the fix also pulls today's daily snapshot from EPPO's oil-share page where Diesel B7 is identified by the `oil_name6v2.png` row label.

fix:
  1. Update `EPPO_URL` to the new index page (kept for documentation/observability; no longer used for navigation).
  2. Add `EPPO_P09_URL` -- direct link to the P09.xls retail-Bangkok monthly file.
  3. Add `EPPO_OIL_SHARE_URL` -- direct link to the daily oil-share snapshot page.
  4. New `_parse_p09_workbook(content)` -- walks col 0 of the "2003-current" sheet to track current year + month; emits one row per WT.AVG bucket (col index 8 = HSD B7); date is YYYY-MM-01.
  5. New `_scrape_oil_share_today()` -- fetches the oil-share PHP page, finds the `oil_name6v2.png` label (Diesel B7), grabs the first numeric `oil_price_colum` value after it (HTML comments stripped first). Returns float or None on parse failure (soft).
  6. Rewrite `_scrape_eppo()` -- downloads P09 (raises on failure), parses it, optionally appends today's daily snapshot, merges with existing seed CSV. Only rows whose date is strictly AFTER the seed max are appended (preserves daily history). Dedup on date.
  7. Add `xlrd==2.0.2` to `requirements.txt` for legacy .xls support (P09 is BIFF .xls, not .xlsx).
  8. New tests: `TestScrapeEppoInternals` (5 tests covering oil-share parser, P09 parser, merge logic, oil-share-failure-graceful path).
