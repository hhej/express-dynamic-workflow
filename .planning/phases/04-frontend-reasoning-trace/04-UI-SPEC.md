## Visual Hierarchy

**Primary focal point:** the live reasoning trace panel (right rail, 384px, D-03) — always-visible, live-streaming steps are the core grading lever and the product's differentiator (CLAUDE.md Core Value: "visible reasoning is what makes this agentic, not just automated"). The chat column is the input/output channel; the trace panel is the *evidence surface*. When the layout is built, the trace panel must read as a first-class peer to the chat column, not a secondary side-pane — full-height, no collapse-by-default on desktop, and the live `running`-state badge animation (Pattern 7 `animate-pulse`) is what draws the eye during query execution.

**Secondary anchor:** the chat input row (bottom of center column) — the user's only persistent action surface. The accent-colored send button (`bg-blue-600`) is the single brightest element on screen at rest, per the accent reserved-for list.

**Tertiary:** the sidebar (left, 256px) — read-only conversation list, low-contrast at rest (`bg-gray-50`), highlights only on active thread. Intentionally recedes.

## Spacing Scale

Declared values (all multiples of 4, mapped to Tailwind v4 default scale):

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Inline gaps inside the trace step status badge, between thumbs-up / thumbs-down buttons |
| sm | 8px | `gap-2`, `p-2` | Compact gaps inside the cap callout (icon ↔ text), between adjacent trace step list items, padding inside JSON `<pre>` blocks |
| md | 16px | `gap-4`, `p-4` | Default panel padding (sidebar, trace panel, chat column inner padding), gap between dashboard cards |
| lg | 24px | `gap-6`, `p-6`, `space-y-6` | Section padding inside the dashboard view, vertical gap between the message list and the chat input |
| xl | 32px | `gap-8`, `p-8` | Empty-state container padding (D-09 explainer + example prompts), top margin between dashboard chart cards |
| 2xl | 48px | `gap-12` | Reserved for future major section breaks (currently unused in Phase 4 layout — trace panel and sidebar are dense by intent) |
| 3xl | 64px | `gap-16` | Reserved for future page-level spacing (currently unused — single-route product per D-01) |

**Layout column widths (D-01 desktop three-column):**

| Column | Width | Tailwind class |
|--------|-------|----------------|
| Sidebar (left, D-02) | 256px fixed | `w-64` |
| Chat column (center) | flex-1 (fills remaining) | `flex-1` |
| Trace panel (right, D-03) | 384px fixed | `w-96` |

**Mobile breakpoint (D-05):** below `md` (768px Tailwind default), columns collapse to chat-only; sidebar and trace panel become drawers.

**Exceptions:** none. The two reserved tokens (2xl, 3xl) are declared so the planner has a documented escape hatch but does not need to use them in Phase 4.

## Typography

Declared sizes and weights (3 sizes, 2 weights — meets the 3–4 sizes / 2 weights cap):

| Role | Size | Weight | Line height | Tailwind class |
|------|------|--------|-------------|----------------|
| Caption / mono detail | 12px | 400 (regular) | 1.5 | `text-xs font-normal leading-normal` (or `font-mono` on the trace step counter and JSON panes per Pattern 7) |
| Body / default | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` (chat messages, trace headlines, sidebar previews, chart axis labels) |
| Heading / emphasis | 16px | 600 (semibold) | 1.2 | `text-base font-semibold leading-tight` (panel titles "Reasoning trace", "Conversations", chart titles, capped callout body, status card headings for `clarify` and `partial`) |

**Notes:**
- The chat surface intentionally runs at 14px body — the locked GFM markdown table (D-10) is denser than typical prose, and 14px keeps the 4-row breakdown table from dominating the column.
- `react-markdown` prose container uses `prose prose-sm max-w-none` (per RESEARCH.md Pattern 4) which inherits the 14px body and applies a consistent vertical rhythm to the LLM-emitted prose without overriding the size scale.
- No display size is declared — Phase 4 has no marketing surfaces or hero sections (single-route product per D-01). If a future phase adds them, declare a 4th size at that time.
- `font-mono` is applied via Tailwind utilities at the same 12px size — it is not a separate size token.

**Voice and tone rules:**
- Use second-person ("you") in instructional copy. Never first-person.
- Use sentence case for headings and buttons (e.g., "New conversation", not "New Conversation").
- Error copy follows the pattern "What happened. What you can do." (e.g., "Couldn't reach the server. Check your connection and try again.")
- "Bangkok Metro" is the user-facing region phrase. The 6 covered provinces (Nonthaburi, Pathum Thani, Samut Prakan, Nakhon Pathom, Samut Sakhon, Ayutthaya) may appear in example prompts and clarification copy.

## Interaction Contracts

Behaviors that the executor must preserve verbatim — these are the "this is how the product works" rules, not the visuals.

| Interaction | Contract |
|------------|----------|
| Chat send (CTA fires) | Disable input + send button → dispatch START → POST /api/chat → live-append trace events to right rail (D-06) → render answer per D-12 status → re-enable input on `done` (D-19) |
| Trace step expand/collapse (D-07) | Click toggles `aria-expanded`; reveals tool/input/output/timestamp; keyboard `Enter`/`Space` activate; arrow keys do NOT navigate between steps (Phase 4 a11y is "reasonable", not full audit per CONTEXT.md Discretion) |
| New conversation (D-14) | Clear `thread_id` from localStorage → reset `useChatStream` reducer → next POST /api/chat omits thread_id → server's first `meta` event sets the new thread_id (D-20) |
| Resume conversation (D-14) | Click thread in sidebar → GET /api/conversations/:id → replay messages into `MessageList` → set `thread_id` in localStorage → next POST /api/chat continues (D-20). Trace panel does NOT swap to past trace (D-08 — past-turn trace deferred to Phase 5) |
| Capped result (D-11) | When `final_payload.capped === true`: strip leading `> ⚠ Cap/floor applied — review recommended\n` from markdown → render `CapCallout` above the cleaned markdown. NEVER both. NEVER invent the banner from `capped` flag alone while leaving the `>` line in (Anti-pattern call-out in RESEARCH.md §Anti-Patterns) |
| Feedback vote (D-17) | Click → set local state to voted → log `{thread_id, message_id, score}` to console + localStorage → buttons remain visible but disabled (`aria-pressed=true`). NO API call. Phase 5 wires API-05. |
| Range toggle (D-15.1) | Click 7d/30d/90d → re-fetch `GET /api/fuel-prices?days=N` → update chart. Default 30d on mount. |
| SSE error (D-19) | `error` event from server → render error toast with `payload.message` → preserve trace entries that arrived before the error → flip input back to enabled |
| Mobile breakpoint (D-05) | <768px (`md`): hide sidebar + trace panel rails → show hamburger drawer for sidebar (left edge) and "Show reasoning" drawer for trace panel (right edge) → focus management restores to triggering button on close |
