# Phase 999.9 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest 8.4.2 (backend) + vitest 4.1.5 (frontend) |
| **Config file** | `pyproject.toml` `[tool.pytest.ini_options]` (backend) / `frontend/vitest.config.*` (frontend) |
| **Quick run command** | `cd backend && pytest tests/test_<changed>.py -x` / `cd frontend && npm test -- --run <changed>.test.tsx` |
| **Full suite command** | `cd backend && pytest -q && cd ../frontend && npm test -- --run` |
| **Estimated runtime** | Backend quick: ~10s · Frontend quick: ~5s · Backend full: ~60s · Frontend full: ~30s · Combined full: ~3-5 min |

## Sampling Rate

- **After every task commit:** Run quick run command for changed file
- **After every plan wave:** Run full suite command (backend + frontend)
- **Before `/gsd:verify-work`:** Full suite must be green (300+ backend tests + 122+ frontend tests, plus ~30+ new from this phase)
- **Max feedback latency:** 60 seconds for per-task quick runs
