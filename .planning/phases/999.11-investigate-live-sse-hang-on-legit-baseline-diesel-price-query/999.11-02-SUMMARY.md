## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] `${EXTRA_ARGS[@]}` tripped `set -u` "unbound variable" in run_5x.sh**
- **Found during:** Task 1 step 1 (baseline run, zero extra args)
- **Issue:** `run_5x.sh` uses `set -u` (strict mode); the inherited bash 3.x on macOS treats `${EXTRA_ARGS[@]}` as unbound when the array has zero elements, causing immediate exit before any probe could run. Plan 01 had not exercised the baseline (zero-arg) path because Plan 01's Task 2 acceptance test forwarded `--warmup-first` (a non-empty case).
- **Fix:** Replaced with bash safe-expansion idiom `${EXTRA_ARGS[@]+"${EXTRA_ARGS[@]}"}` — array is dereferenced only when non-empty. Behaves identically in the non-empty case; expands to nothing in the empty case.
- **Files modified:** `.planning/phases/999.11-.../repro/run_5x.sh`
- **Verification:** Step 1 ran 5/5 without spurious exit; step 2-4 (non-empty arg cases) also ran cleanly, proving the fix is backward-compatible.
- **Committed in:** `6831be1` (rolled into the evidence-gathering commit since it is a precondition to gathering step 1)
