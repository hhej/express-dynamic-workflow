## Conclusion

Hypothesis (b) is CONFIRMED. The bug is NOT in the `tool_call_count` `operator.add` reducer (that mechanism remains correct and is being defensively pinned in Task 2). The bug is in `planner_node`'s cache-aware override cascade (planner.py:500-509): when fuel is fresh and route is missing, the cascade unconditionally promotes `next_step` to `fetch_route` without checking whether `merged_destination` is non-None.

**Live failure mechanism (deterministic):**

1. Turn 1: user asks `"What's the current diesel price in Bangkok?"`
2. `guard_input` allows (domain match on "diesel"), no trace entry on the allow path
3. `planner` pass 1: LLM emits `next_step=fetch_fuel`, `destination=None`, `user_intent=surcharge_query` (the question reads as fuel-related; the LLM picks the closest schema next_step)
4. `fuel_agent` runs successfully, populates `state.fuel_data` with `fetched_at=<now>`, emits +1 to `tool_call_count`
5. `planner` pass 2: LLM again emits `next_step=fetch_fuel` (same question, same prompt — LLM is stateless from its own perspective)
6. **Cache-aware override fires:** `_fuel_fresh(state) == True` → check `_route_matches(state, None, None)` → False → set `next_step="fetch_route"` (planner.py:509)
7. `route_agent` runs, raises `ValueError("route_agent_node requires 'destination'")` per Plan 05-09 D-10 selective ValueError catch
8. D-24 `_wrap_error_sink` catches the ValueError (BUT wait — D-09 mandates ValueError bubbles uncaught for pricing; for route_agent, _wrap_error_sink at graph.py:108-110 explicitly re-raises ValueError). The error reaches the SSE stream as an `error` event and the stream emits `done`. Total elapsed ~10s.

**Fix shape (per plan Task 2):** Add a defensive short-circuit in `planner_node` BEFORE the LLM `model.invoke(...)` call. Pre-conditions for the short-circuit: `state.fuel_data is not None` AND `not state.destination` AND `not state.shipping_type` AND `not state.weight_kg` (the legit-baseline shape — fuel asked, no logistics fields). When all four hold, set `next_step="respond"` and emit one observability trace entry. This closes the gap WITHOUT touching the cache-aware override (which is correct for surcharge queries that DO have a destination).

The defense-in-depth reducer test from Task 2 step A lands regardless — it pins the `operator.add` invariant under fan-out so a future state-schema refactor cannot silently regress.
