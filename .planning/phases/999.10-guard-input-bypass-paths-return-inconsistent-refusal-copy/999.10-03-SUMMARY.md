## Decisions Made

- **Removed defensive monkeypatch on `guard_input.get_chat_model`** (PLAN <action> specified it, but `guard_input.py` imports `get_chat_model` lazily inside `_llm_fallback` at line 134 — the attribute does not exist at module level). The defensive guard was unreachable in practice anyway: `GUARD_INPUT_USE_LLM_FALLBACK` defaults False and the regex catches the four representative cases. Removing the monkeypatch keeps the test passing without changing its assertion strength.
- **`test_legit_baseline_does_not_refuse` invokes `planner_node` directly** (not the full graph). Rationale: the legit-vs-refusal fork happens entirely inside `planner_node`, so the unit-level assertion is the correct scope. Going through `build_graph` would require mocking fuel/route/pricing specialist agents, which is out of scope for a false-positive guard.
