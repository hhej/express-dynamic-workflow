<objective>
Fix gap-2 from 05-UAT.md test 4: querying an out-of-Bangkok-Metro destination (e.g. Ayutthaya districts that geocode outside the supported set, or Lop Buri / Chiang Mai) raises an uncaught `ValueError("No Bangkok Metro zone for {destination!r}")` from `calculate_route._zone_for_destination`. The Phase 3 D-23 retry-allow-list deliberately excludes ValueError, so the exception bubbles to the SSE error event channel and the user gets a dead conversation with no recovery path.

The Bangkok Metro rename (quick task 260425-vc6) intentionally shrank zone coverage from the original Central Region scope to a Bangkok-Metro-only set. The shrink was correct per backlog 999.2 option (b), but `route_agent_node` was never updated to handle the corresponding zone-miss case gracefully, AND the user-facing docs still imply broader coverage.

Two-part fix per UAT remediation_hint:
1. **Code:** `route_agent_node` catches `ValueError` whose message starts with `"No Bangkok Metro zone"` (NOT every ValueError — D-10 contract failures from missing origin/destination still bubble per Phase 3 D-09), appends a structured entry to `state.errors`, returns `next_step="respond"`. The Phase 3 D-24 planner-short-circuit-on-errors path then routes to response_node, which renders status='partial' prose via the existing `_render_prose_partial(state)` helper. No response_node change required — leverage the established error-sink contract.
2. **Docs:** README.md, docs/data-sources.md, and docs/architecture.md gain explicit Bangkok-Metro province enumeration AND list the provinces dropped from the original Central Region. Restores DOC-01 / DOC-02 / DOC-04 honesty.

Purpose: restore Phase 5 ROADMAP success criterion #2 (HITL trigger via central-3 query — UAT test 4 was meant to demonstrate HITL via Ayutthaya, but the route tool crashed before HITL could trigger; the gap-2 doc fix also clarifies which provinces ACTUALLY produce central-3 routes for HITL demos) AND ROADMAP success criterion #5 (docs accurate for submission). Restores ORCH-08 (graceful fallback) and ORCH-03 (Route Agent's contract: never crash the agent loop on a tool error that has a user-recoverable cause).
