<objective>
Implement ORCH-07 by extending the existing `_route_from_planner` conditional edge in `backend/agent/graph.py` to return a list `["fuel_agent", "route_agent"]` (instead of a single string) when BOTH fuel_data is stale AND route_data is missing-or-mismatched. Per LangChain docs, a list-returning conditional edge schedules every node in the same Pregel superstep — achieving the parallel execution that ORCH-07 requires WITHOUT introducing the dynamic-payload Send API (which would force fabricating per-branch state slices for no benefit since both nodes read the same state keys).

The `operator.add` reducer on `reasoning_trace` and `errors` (Phase 2 Pitfall 1, Phase 3 D-05) already handles concurrent appends. `fuel_data` and `route_data` are scalar dict keys written by disjoint branches — last-write-wins is safe. No new reducers introduced (D-02).
